## 1. 冲突超图 H_cf 的精确定义

- **顶点**：基本象限的全部 m² = 1369 个候选 cell。`V = {0,…,m−1}²`。
- **(X) 超边（三元，97% 的冲突）**：同一共线 C4-lift 直线上的 3 个 cell 组成的三元组。
  即 `line_cons` 中每条直线 L 的所有 3-子集。rank = 3。
- **(S) 边（二元，~3%）**：某直线上有 ≥2 个 lifted 点的 cell 与该直线上另一 cell
  组成的 2-元组（slope±1 约束）。rank = 2。
- 一个 rot4 配置（cell 集 C）合法 ⇔ C 是 H_cf 的**独立集**（不含任何 (X) 超边、
  且每条 (S) 线至多选 2 个）。

> 注意：H_cf 是 **几何冲突超图**，定义在全部 m² 候选 cell 上（FULL）。诱导在典型
> 2-因子上的子图（INDUCED）稀疏得多（见 §3），但定理的 (C2)/(C3) 条件看的是 **FULL**
> 超图的度数/codegree。

## 2. 定理：冲突有界条件（Graves / 2407.18144, Theorem 3.1）

给定宿主超图 H（其匹配即我们要选的对象），冲突族 C 称为 **(d, ℓ, ε)-bounded**，若：

- **(C1)** `2 ≤ |C| ≤ ℓ` 对所有 C∈C；
- **(C2)** `Δ(C^(j)) ≤ ℓ·d^{j−1}`，对所有 `j ∈ [2, ℓ]`；
- **(C3)** `Δ_{j′}(C^(j)) ≤ d^{j−j′−ε}`，对所有 `j ∈ [2, ℓ]`、`j′ ∈ [2, j−1]`；
- （(C4)(C5) 为邻域码度，对本文的直接验证可由 (C3) 推出，略。）

其中 `d` 是宿主超图的度参数（我们取 cell 级可用度 `d = |V| = m²`），`ℓ` 是冲突最大
rank（此处 `ℓ = 3`），`ε ∈ (0,1)` 是自由常数。若 H 自身满足 Pippenger codegree
条件 `(H1)(H2)`，则存在覆盖全部指定顶点、且 C∪D-free 的匹配。

**对本文最关键的是 (C2)(C3)**：它们把"冲突能否被避开"完全归结为冲突超图的
**度数**和**codegree**相对于宿主度 `d` 的大小——这正是用户说的"控制 codegree 而非
总度数"，也是它越过 LLL 死胡同（LLL 看总度数 ~千万级）的机理。

## 4. (C2)(C3) 精确核对（host d = 1369, ℓ = 3）

### (C2)
- j=3：`deg3 ≤ ℓ·d² = 3·1369² = 5,622,483`。
  FULL `deg3_max = 92,488` ≪ 5.6e6 ⇒ **OK**（即使取 FULL 最大度数也满足）。
- j=2：`deg2 ≤ ℓ·d = 3·1369 = 4,107`。
  `deg2_max = 288` ≪ 4,107 ⇒ **OK**。

### (C3) （决定生死的一步：codegree 必须 ≪ d）
- j=3, j′=2：要求 `co3 ≤ d^{1−ε}`，对若干 ε 取值：

| ε | 上界 `d^{1−ε}` | `co3_max = 1,132` | 判定 |
|---|---|---|---|
| 0.5 | 37.0 | 1,132 | **FAIL** |
| 0.3 | 156.8 | 1,132 | **FAIL** |
| 0.2 | 322.9 | 1,132 | **FAIL** |
| 0.1 | 664.9 | 1,132 | **FAIL** |
| 0.05 | 954.1 | 1,132 | **FAIL** |
| →0 (极限) | →1369 | 1,132 | 临界通过（仅当 ε ≲ 0.026） |

> **关键判据**：`co3_max = 1,132` 介于 `d^{0.95}=954` 与 `d=1369` 之间。它在标准
> 常数 ε（≥0.05）下**全部 FAIL**；仅当 ε ≲ 0.026（上界趋近 `d=1369`）时才"临界通过"。
> 但 ε 是定理 `(d,ℓ,ε)`-bounded 定义中的**固定常数**，且同时出现在 `(H2)`、
> 顶点数界 `(S2): d^ε ≤ |P| ≤ exp(d^{ε³})` 等辅助条件中。取 ε→0 时这些辅助界在
> **有限 d=1369** 下不再成立（例如 `exp(d^{ε³})` 当 ε=0.02 时仅 ≈2.9 ≪ |P|=1369）。
> 因此 cell 级尺度下 (C3) **不能在本有限实例被定理黑箱满足**。

## 5. 结论（已用 `codegree_m37.py` 实测值定稿）

### 5.1 实测 verdict
- **(C2) 全部通过**：`deg3_max=92,488 ≪ 5.6e6`，`deg2_max=288 ≪ 4,107`。
- **(C3) 在 cell 级尺度失败**：`co3_max=1,132` 介于 `d^{0.95}=954` 与 `d=1369` 之间，
  对标准 ε≥0.05 全部 FAIL；仅当 ε≲0.026 临界通过，但此时定理的辅助界（顶点数、H2）
  在有限 d=1369 下崩塌。

### 5.2 根本性质：Graves / 2407.18144 是 **渐近定理（d→∞）**
定理的 (C2)(C3) 及 (H1)(H2)(S2) 条件中 d 的 ε 次幂、指数界，是为 **d 充分大** 设计的。
在 m=37 这个**有限**实例，`d=1369` 不够大，导致：
- 它**能证明"所有足够大的 m 存在冲突自由 2-因子"**（co3 随 m 增长慢于 d=m²，故对大 m
  (C3) 必然成立）；
- 但**不能黑箱证明 m=37**——m=37 恰好落在渐近阈值**之下**，其实测 codegree 1132 是
  "临界"值（仅略低于 d）。

### 5.3 但这恰恰确认了方向 1 的正确性（匹配/nibble 是正确路线）
与 LLL 对比：LLL 的 dependency 度 `d ≤ 16,958,430`（千万级），而冲突超图 codegree 仅
`1,132`——**小了约 4 个数量级**。这正是用户说的"控制 codegree 而非总度数"：LLL 因看
总度数而死的死胡同，被 codegree 路线完全绕开。本计算把这一点坐实成了可核对的数字。

### 5.4 为什么 m=37 仍应由 (b) nibble 闭合（而非本定理黑箱）
- 定理给的是**非构造**存在性，且仅对大 m 生效；m=37 需要**构造性**方法。
- 实际 nibble 面对的是 **INDUCED** 冲突超图（典型 2-因子上 `avg_x_deg_per_cell=157`、
  `x_deg_max=380`），其 codegree 远小于 FULL 的 1132（诱导 cell-对 codegree 期望
  ≈ 157/37 ≈ 4 量级），**稳稳落在安全区**。nibble 逐格挑选、只与已选小集比较，分钟级
  可运行，且一旦成功即真找到一个 m=37 解（闭合问题）。
- 因此 (c) 的最终价值：**把抽象定理落成 m=37 上可 verify 的具体不等式，并证明"匹配/
  nibble 路线在 codegree 上完全可行（比 LLL 小 10⁴×），m=37 仅因处于渐近阈值之下而需
  改走构造性 nibble"**。

### 5.5 诚实边界（理论限制，已写入 research_H.md §5）
1. "整 2-因子 = 一条超边（rank k=74）"直接套 Pippenger `(H2): Δ₂(H)≤Δ/(2k²)` 会失败：
   兼容 cell 在同一 2-因子中的共现比例是线性 `1/(2m−2)≈1/72`，而 Pippenger 要 `1/k²`。
   故 Graves 定理**不能在整结构编码下直接给存在证明**；其 codegree 思想只在 cell 级
   nibble 尺度（本文 §2–§4）真正起作用。
2. 本定理对 m=37 是**临界/渐近外**实例，不能黑箱认证；闭合靠 (b) nibble（构造性）。
